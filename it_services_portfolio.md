# Каталог IT-услуг: пакетные решения

## Категория 1: Разработка сайтов и веб-приложений

### 1. Landing Page «под ключ» для лидогенерации
**Для кого:** запуск нового продукта, тестирование ниши, подготовка к рекламным кампаниям.
**Краткое описание:**
Одностраничный сайт с высокой конверсией под контекстную и таргетированную рекламу. Фокус на один оффер и одно целевое действие.

**Техническая спецификация:**
*   **Стек:** JAMstack (Next.js 15, Astro, Nuxt 3) со статической генерацией для мгновенной загрузки, либо headless-CMS (Payload, Strapi) под частые правки.
*   **Дизайн:** индивидуальный UI/UX без шаблонов, Mobile First, адаптация под экраны высокого разрешения.
*   **Интеграции:** передача лидов в CRM (amoCRM, Bitrix24, RetailCRM) через Webhook/API, подключение Яндекс Метрики и пикселя VK Рекламы, настройка целей конверсии.
*   **Производительность:** Lazy Load, изображения в WebP/AVIF, минификация, целевые оценки 90+ в PageSpeed Insights.
*   **Формы:** клиент-серверная валидация, защита от ботов (Yandex SmartCaptcha, hCaptcha, Cloudflare Turnstile), honeypot-поля.

### 2. Корпоративный сайт (B2B/B2C портал)
**Для кого:** компании, формирующие репутацию и собирающие органический трафик.
**Краткое описание:**
Многостраничный сайт с CMS, каталогом услуг, блогом, личным кабинетом и сложной информационной архитектурой.

**Техническая спецификация:**
*   **Стек:** 1С-Битрикс, WordPress — для классического подхода; headless (Payload, Strapi, Directus) + Next.js/Nuxt — для кастомного фронта.
*   **Структура данных:** кастомные типы записей под портфолио, команду, отзывы, услуги; настройка отношений между сущностями.
*   **SEO-база:** автогенерация XML-карт, шаблоны мета-тегов, микроразметка Schema.org (JSON-LD), ЧПУ, корректные hreflang и canonical.
*   **Безопасность:** SSL (Let's Encrypt или коммерческий), защита админки от брутфорса (fail2ban, rate limiting), скрытие стандартных путей, строгая CSP.
*   **Администрирование:** визуальный конструктор страниц, ролевая модель (редактор/главред/администратор), аудит-журнал правок.

### 3. Интернет-магазин (E-commerce Basic)
**Для кого:** SMB с ассортиментом до 10 000 позиций, ищущие быстрый запуск продаж.
**Краткое описание:**
Готовый интернет-магазин с каталогом, корзиной, онлайн-оплатой и базовой логистикой.

**Техническая спецификация:**
*   **Платформа:** Medusa.js, 1С-Битрикс, WooCommerce, InSales, Tilda Store — выбор под задачи клиента и штат.
*   **Каталог и поиск:** иерархический каталог, атрибуты и варианты (цвет, размер), асинхронные фильтры, поиск с учётом морфологии русского языка.
*   **Чекаут и платежи:** одностраничный чекаут, ЮKassa, Т-Касса, CloudPayments, Robokassa, Сбербанк Эквайринг; поддержка СБП; онлайн-касса по 54-ФЗ (АТОЛ Онлайн, Orange Data, Ferma).
*   **Логистика:** интеграция с СДЭК, Почтой России, Boxberry, Яндекс Доставкой, DPD — автоматический расчёт стоимости и сроков.
*   **Уведомления:** email через UniSender/Mailgun/SendPulse, SMS через SMS.ru/SMSAero/Exolve, Telegram-уведомления менеджеру.

### 4. E-commerce платформа Enterprise
**Для кого:** маркетплейсы, крупные ритейлеры, B2B-платформы с высокой нагрузкой и сложными бизнес-правилами.
**Краткое описание:**
Масштабируемый интернет-магазин с B2B-кабинетами, сложными программами лояльности и глубокой интеграцией с учётными системами.

**Техническая спецификация:**
*   **Архитектура:** микросервисы или модульный монолит на Node.js/Go/Java/Python, фронтенд с SSR (Next.js, Nuxt, Remix).
*   **Кэширование и поиск:** Redis/KeyDB для сессий и корзин, Elasticsearch/OpenSearch/Meilisearch для полнотекстового поиска с учётом опечаток и морфологии.
*   **Интеграции ERP/CRM:** двусторонний обмен с 1С:ERP, 1С:УТ, SAP, Microsoft Dynamics по защищённым API — синхронизация остатков, цен, контрагентов, заказов в реальном времени.
*   **Инфраструктура:** Docker + Kubernetes, балансировщики нагрузки (Nginx, HAProxy, Traefik), Primary-Replica репликация PostgreSQL/MySQL, автомасштабирование.
*   **Промо-движок:** накопительные программы лояльности, динамические промокоды, сегментные мультипрайсы, сложные акционные комбинации.

### 5. Веб-приложение (SPA / SaaS)
**Для кого:** продуктовые команды, запускающие SaaS-сервисы, кастомные CRM, дашборды аналитики, личные кабинеты.
**Краткое описание:**
Интерактивный сервис, работающий без перезагрузки страниц, с логикой уровня нативного приложения.

**Техническая спецификация:**
*   **Frontend:** React/Vue/Svelte + современный state management (Redux Toolkit, Zustand, Pinia, TanStack Query).
*   **Backend:** REST или GraphQL на Node.js (NestJS, Fastify), Go, Python (FastAPI, Django), Java (Spring Boot), .NET.
*   **Аутентификация:** JWT, OpenID Connect поверх OAuth 2.0 (вход через Яндекс ID, VK ID, T-ID, Сбер ID), двухфакторная аутентификация (TOTP, SMS, email).
*   **Real-time:** WebSockets или Server-Sent Events для чатов, статусов, live-обновлений данных.
*   **Тестирование:** Unit/Integration/E2E тесты (Vitest, Jest, Playwright, Cypress), CI/CD-пайплайны.

## Категория 2: Мобильная разработка

### 6. Нативные iOS и Android приложения
**Для кого:** продукты, требующие максимальной производительности, доступа к аппаратным возможностям и премиального UX.
**Краткое описание:**
Полноценные нативные приложения на Swift/SwiftUI и Kotlin/Jetpack Compose с публикацией в App Store, Google Play и RuStore.

**Техническая спецификация:**
*   **iOS:** Swift 5+, SwiftUI/UIKit, архитектура MVVM или TCA (The Composable Architecture), Combine и async/await.
*   **Android:** Kotlin, Jetpack Compose/XML, архитектура MVVM или MVI, Coroutines + Flow.
*   **Хранилища:** Core Data/SwiftData, Room; шифрование чувствительных данных через Keychain/Keystore.
*   **Push-уведомления:** APNs для iOS, FCM для Android (GMS), RuStore SDK для российских сторов, HMS Push для AppGallery.
*   **Публикация:** прохождение модерации App Store, Google Play, RuStore, AppGallery; оформление страниц, скриншоты, ASO-оптимизация.

### 7. Кросс-платформенное приложение (React Native / Flutter)
**Для кого:** стартапы и SMB, которым нужен быстрый выход на iOS и Android с единой кодовой базой.
**Краткое описание:**
Одно приложение для двух платформ с близким к нативному UX при экономии 30-50% бюджета.

**Техническая спецификация:**
*   **Стек:** React Native (Expo или bare workflow) либо Flutter 3+ — выбор под команду и требования к производительности.
*   **Навигация и состояние:** React Navigation + Redux/Zustand или Flutter Router + Riverpod/Bloc.
*   **Нативные модули:** написание bridge-модулей под специфичные задачи (сканеры, BLE, платежи, биометрия).
*   **CI/CD:** сборка через Fastlane, EAS Build, Codemagic; автоматическая публикация бета-версий через TestFlight и Google Play Internal Testing.
*   **Мониторинг:** Sentry для ошибок, AppMetrica или Firebase Crashlytics для аналитики и крешей.

### 8. Публикация, обновление и ASO-оптимизация
**Для кого:** владельцы существующих приложений, которым нужна регулярная публикация обновлений и рост в сторах.
**Краткое описание:**
Выделенная услуга сопровождения мобильных приложений в магазинах без привязки к разработке.

**Техническая спецификация:**
*   **Публикация:** подготовка сборок под требования App Store Review Guidelines, Google Play Policy, RuStore Guidelines, AppGallery Review; прохождение модерации и обжалование отказов.
*   **ASO:** анализ ключевых запросов через ASOdesk, AppFollow, AppTweak; оптимизация названия, подзаголовка, описания, ключевых слов; A/B-тестирование иконок и скриншотов через App Store Connect и Google Play Console.
*   **Обновления:** релизные циклы, фазированная раскатка, управление версиями и обратной совместимостью.
*   **Аналитика:** подключение AppMetrica, Firebase, AppsFlyer, Adjust для атрибуции установок и поведенческой аналитики.

## Категория 3: Чат-боты и ИИ

### 9. Чат-бот-лидогенератор
**Для кого:** компании, собирающие заявки в мессенджерах; замена или дополнение лендинга.
**Краткое описание:**
Бот для первичного контакта с клиентом, квалификации лида и передачи его в CRM.

**Техническая спецификация:**
*   **Платформа:** Telegram Bot API, VK Callback API, WhatsApp Business API, Max, Avito Messenger API; библиотеки aiogram, grammY, Telegraf.
*   **Функционал:** многоуровневые меню, inline-кнопки, квиз-воронки с ветвлением, сбор контактных данных.
*   **Хранение:** PostgreSQL или MongoDB для профилей и истории диалогов.
*   **Уведомления:** мгновенная передача лидов менеджерам в Telegram/email, автоматическая маршрутизация в amoCRM, Bitrix24, RetailCRM через Webhook.
*   **Хостинг:** Webhook-архитектура на выделенном VPS или облачных функциях (Yandex Cloud Functions, Serverless Containers).

### 10. Telegram Mini App (магазин внутри мессенджера)
**Для кого:** бренды, продающие через Telegram, и сервисы, которым нужен быстрый доступ без установки приложения.
**Краткое описание:**
Полноценный интерфейс внутри Telegram с каталогом, корзиной и платежами — без перехода в браузер.

**Техническая спецификация:**
*   **Технология:** Telegram Web Apps SDK, адаптация под нативную тему мессенджера, работа с Telegram.WebApp API.
*   **Взаимодействие с ботом:** двусторонний обмен через initData, криптографическая валидация подписи на сервере.
*   **Корзина и платежи:** Telegram Payments с провайдерами ЮKassa, Т-Касса, CloudPayments (для РФ); Stripe для международных направлений; формирование инвойсов.
*   **Админка:** отдельная панель управления каталогом, заказами и статистикой продаж.
*   **Безопасность:** проверка hash initData на сервере, защита от подделки user_id и суммы заказа.

### 11. Бот-автоматизатор техподдержки (HelpDesk)
**Для кого:** компании с высоким потоком типовых обращений, которые перегружают операторов.
**Краткое описание:**
Система первой линии поддержки: бот отвечает на частые вопросы и маршрутизирует сложные тикеты специалистам.

**Техническая спецификация:**
*   **Поиск и анализ:** нечёткий поиск (Elasticsearch, Meilisearch) и семантический поиск на векторных базах (Qdrant, Weaviate, pgvector) — в связке.
*   **Тикеты:** генерация ID, статусы, приоритеты, SLA-таймеры; интеграция с Usedesk, Omnidesk, HappyFox, Jira Service Management.
*   **Рабочее место оператора:** встроенная панель или интеграция с существующим HelpDesk/CRM.
*   **Аналитика:** CSAT, FRT, среднее время решения, процент самообслуживания.
*   **Эскалация:** автоматические триггеры по SLA — уведомления руководителю при превышении таймеров.

### 12. ИИ-ассистент для бизнеса (LLM + RAG)
**Для кого:** компании с большой базой знаний, которые хотят автоматизировать консультации без потери качества.
**Краткое описание:**
LLM-ассистент, обученный на корпоративных данных, консультирующий клиентов и сотрудников на уровне профильного специалиста.

**Техническая спецификация:**
*   **Модели:** Claude Opus 4 / Sonnet 4, GPT-4o через API; YandexGPT, GigaChat для российского контура; локальные open-source модели (Llama, Qwen, DeepSeek, Mistral) для закрытых инсталляций.
*   **RAG-архитектура:** парсинг корпоративной базы (сайт, документы, Confluence, Notion, 1С), генерация эмбеддингов, хранение в Qdrant/pgvector/Weaviate, поиск релевантного контекста.
*   **Управление контекстом:** контроль лимитов токенов, кеширование системных промптов, версионирование «личности» ассистента.
*   **Мультиканальность:** виджет на сайте, Telegram, WhatsApp, VK, Max, интеграция с CRM.
*   **Guardrails:** защита от prompt injection, ограничение тем диалога, фильтрация «галлюцинаций», модерация вывода.

### 13. Бот-букинг (запись на услуги)
**Для кого:** салоны красоты, клиники, автосервисы, коворкинги, спортзалы.
**Краткое описание:**
Автоматическая запись клиентов к специалисту на свободный слот с синхронизацией с реальным расписанием.

**Техническая спецификация:**
*   **Интеграции:** YClients, Altegio, Dikidi, Google Calendar, Яндекс Календарь; либо собственная логика бронирования.
*   **Обработка времени:** часовые пояса, длительность комбинированных услуг, предотвращение overbooking, графики и перерывы специалистов.
*   **Уведомления:** каскадные напоминания о визите, сбор отзывов через 1-2 дня после визита.
*   **Финансы:** предоплата через СБП/ЮKassa/Т-Касса, холдирование средств для снижения no-show.

## Категория 4: Интеграции с учётными и российскими системами

### 14. Внедрение CRM (amoCRM, Bitrix24, RetailCRM)
**Для кого:** отделы продаж, теряющие лиды из-за хаоса в каналах коммуникации.
**Краткое описание:**
Настройка CRM «в одно окно»: воронки продаж, автоматизация рутины, интеграция всех источников заявок.

**Техническая спецификация:**
*   **Воронки:** проектирование статусов сделок под бизнес-процесс, кастомные поля, сегментация, LTV, источники.
*   **Каналы:** лид-формы сайта, email, Telegram, WhatsApp, VK, Авито, соцсети — единый inbox.
*   **Телефония:** интеграция с Mango Office, UIS, Телфин, Sipuni, MTT; всплывающие карточки клиента при входящем звонке, запись разговоров.
*   **Digital-воронка:** триггеры автосообщений, автопостановка задач, сценарии реанимации.
*   **Ролевая модель:** разграничение доступа, контроль выгрузки базы, аудит действий менеджеров.

### 15. Корпоративный портал (Bitrix24, ELMA365, SimpleOne)
**Для кого:** компании от 50 сотрудников, которым нужна единая точка для коммуникации, задач и документов.
**Краткое описание:**
Внутренняя платформа для проектов, чатов, задач и электронного согласования.

**Техническая спецификация:**
*   **Бизнес-процессы:** визуальные редакторы маршрутов согласования (отпуска, счета, договоры) с многоуровневыми условиями.
*   **Экстранет:** защищённые проектные группы для работы с внешними подрядчиками и клиентами.
*   **Интеграции:** обмен данными с 1С:ЗУП, 1С:Бухгалтерия, SAP; интеграция с корпоративной почтой, календарями, системами ЭДО (Диадок, СБИС, Такском).
*   **Автоматизация:** учёт активов (оборудование, пропуска, транспорт), автогенерация типовых договоров и актов по шаблонам.

### 16. Интеграция с 1С и ERP
**Для кого:** ритейл, оптовая торговля, производство — все, кто ведёт учёт в 1С.
**Краткое описание:**
Двусторонняя синхронизация 1С с сайтом, CRM, маркетплейсами и внешними сервисами.

**Техническая спецификация:**
*   **Протоколы:** штатный обмен CommerceML (XML), REST API 1С, HTTP-сервисы, OData, Web-сервисы SOAP.
*   **Каталог:** многоскладовость, разные типы цен (опт/розница/клиентские), древовидные категории, атрибуты, медиафайлы.
*   **Заказы и контрагенты:** двусторонняя передача заказов, синхронизация статусов, актуализация контрагентов и договоров.
*   **Оптимизация:** пакетный обмен, дельта-обмен (только изменённые объекты), очереди (RabbitMQ, Kafka) для снижения нагрузки.
*   **Мониторинг:** подробное логирование, алерты по расхождениям артикулов, остатков и дублям.

### 17. Интеграция с маркетплейсами (Wildberries, Ozon, Яндекс Маркет, Мегамаркет)
**Для кого:** продавцы, работающие одновременно с несколькими площадками.
**Краткое описание:**
Централизованное управление товарами, остатками, ценами и заказами через API маркетплейсов.

**Техническая спецификация:**
*   **API:** прямая работа с API Wildberries, Ozon Seller, Яндекс Маркет Partner API, Мегамаркет, Lamoda.
*   **Каталог:** массовая выгрузка и обновление карточек, управление контентом, сопоставление атрибутов площадок.
*   **Остатки и цены:** синхронизация с 1С/ERP в реальном времени, защита от переплаты за overselling (продажа большего количества, чем есть на складе).
*   **Заказы:** приём заказов в единый инбокс, автоматическая маршрутизация по FBS/FBO схемам, работа с возвратами.
*   **Аналитика:** отчёты по продажам по площадкам, маржинальности, оборачиваемости; интеграция с BI.

### 18. API-шлюзы и middleware
**Для кого:** компании с разрозненным IT-ландшафтом, где системы не общаются напрямую.
**Краткое описание:**
Промежуточный слой, связывающий несовместимые системы, базы и сторонние сервисы.

**Техническая спецификация:**
*   **Стек:** Node.js (NestJS, Fastify), Go, Python (FastAPI); контейнеризация для масштабирования.
*   **Трансформация:** маппинг и конвертация XML/JSON/CSV «на лету» под требования принимающей стороны.
*   **Очереди:** RabbitMQ, Kafka, Yandex Message Queue — асинхронная обработка больших массивов, гарантия доставки при отказах.
*   **Безопасность:** Rate Limiting, валидация схем (JSON Schema, Zod), API Keys, OAuth 2.0, mTLS для внутренних интеграций.

### 19. Платёжные интеграции и фискализация
**Для кого:** интернет-магазины, подписочные сервисы, платформы с любым видом онлайн-оплат.
**Краткое описание:**
Подключение российских и международных платёжных систем, рекуррентных списаний и онлайн-касс по 54-ФЗ.

## Категория 5: DevOps и серверная инфраструктура

### 20. Развёртывание и тюнинг выделенных серверов
**Для кого:** проекты, выходящие с shared-хостинга или переезжающие с одного провайдера на другой.
**Краткое описание:**
Подготовка VPS/Dedicated под production: стабильность, скорость, безопасность.

**Техническая спецификация:**
*   **Провайдеры:** Selectel, Yandex Cloud, VK Cloud, MTS Cloud, Timeweb Cloud, Beget; для экспорта — Hetzner, DigitalOcean, AWS.
*   **ОС и окружение:** Ubuntu LTS, Debian, AlmaLinux, Astra Linux, РЕД ОС (для импортозамещения); веб-стек (Nginx + backend + БД) или контейнеризация (Docker, Podman).
*   **Сетевая безопасность:** iptables/nftables, скрытие портов, SSH по ключам без паролей, fail2ban, CrowdSec.
*   **Тюнинг:** настройка Nginx (worker_processes, keepalive, буферы), пулов PHP-FPM/uWSGI/gunicorn, параметров PostgreSQL/MySQL под железо.
*   **Шифрование:** Let's Encrypt через Certbot или acme.sh, коммерческие сертификаты, HSTS и OCSP stapling.
*   **Резервирование:** автоматические бэкапы в S3 (Selectel Object Storage, Yandex Object Storage, VK Cloud Storage), дедупликация через restic/borg.

### 21. CI/CD-конвейер
**Для кого:** команды, деплоящие вручную и теряющие время на ошибках выкатки.
**Краткое описание:**
Автоматизация сборки, тестирования и деплоя — новые версии едут на прод без ручного вмешательства.

**Техническая спецификация:**
*   **Инструменты:** GitLab CI, GitHub Actions, GitVerse CI, Jenkins, Drone, TeamCity — выбор под существующий git-хостинг.
*   **Сборка:** контейнеризация, компиляция ассетов, сборка Docker-образов, публикация в приватные реестры (Harbor, GitLab Registry, Yandex Container Registry).
*   **Автотесты:** статический анализ (линтеры ESLint/Ruff/golangci-lint, SAST-сканеры SonarQube/CodeQL, Semgrep), unit/integration/E2E; блокировка деплоя при падении.
*   **Deploy:** Zero-Downtime стратегии (Blue-Green, Canary, Rolling), Ansible/Terraform для инфраструктуры, ArgoCD/Flux для GitOps в Kubernetes.
*   **Уведомления:** статусы сборки в Telegram, Mattermost, Slack; алерты об упавших пайплайнах.

### 22. Масштабирование Highload-систем
**Для кого:** проекты с пиковыми нагрузками (распродажи, запуски, сезонный трафик) и деградацией производительности.
**Краткое описание:**
Подготовка архитектуры к удержанию тысяч RPS без увеличения времени отклика.

**Техническая спецификация:**
*   **Балансировка:** Nginx, HAProxy, Traefik, Envoy; распределение трафика между пулом App-серверов.
*   **Кэширование:** Redis/KeyDB для сессий и горячих данных, кэш страниц на уровне веб-сервера (FastCGI cache, Varnish), CDN-кэширование.
*   **Базы данных:** Primary-Replica репликация PostgreSQL/MySQL, разделение read/write, профилирование запросов (pg_stat_statements, slow log), шардирование (Citus, Vitess) для сверхбольших таблиц.
*   **CDN:** Cloudflare, Yandex CDN, VK Cloud CDN, CDN-video, Selectel CDN — для статики и видео.

### 23. Мониторинг и алертинг (Observability)
**Для кого:** команды, узнающие о проблемах от клиентов, а не из систем мониторинга.
**Краткое описание:**
Полный стек наблюдаемости: метрики, логи, трейсы и разумная система алертов.

**Техническая спецификация:**
*   **Метрики:** Prometheus + Grafana, VictoriaMetrics, Zabbix — сбор CPU, RAM, диска, сети, бизнес-метрик.
*   **Логи:** ELK/OpenSearch, Loki, Graylog — централизованный сбор, индексация, полнотекстовый поиск.
*   **APM:** Sentry, Grafana Tempo + OpenTelemetry, Elastic APM — трейсинг ошибок до конкретных функций и запросов.
*   **Алертинг:** Alertmanager, Grafana OnCall; каскадные уведомления (Telegram → SMS → звонок) по степени критичности и SLA.

## Категория 6: Безопасность и QA

### 24. Информационная безопасность (DevSecOps и WAF)
**Для кого:** проекты, обрабатывающие персональные данные, платежи и чувствительную информацию.
**Краткое описание:**
Комплексная защита от взломов, DDoS, утечек ПДн и атак на приложение.

**Техническая спецификация:**
*   **Аудит уязвимостей:** сканирование OWASP Top 10 (SQLi, XSS, CSRF/SSRF, IDOR, broken auth), SAST (Semgrep, SonarQube), DAST (OWASP ZAP, Burp), проверка зависимостей (Trivy, Snyk, Dependency-Track).
*   **WAF:** Qrator, DDoS-Guard, StormWall, Nemesida WAF, Cloudflare WAF, ModSecurity; поведенческие правила, фильтрация ботов.
*   **Anti-DDoS:** многоуровневая защита L3/L4/L7, Rate Limiting, challenge-страницы для подозрительного трафика.
*   **Криптография и секреты:** шифрование данных at rest (LUKS, pgcrypto, TDE), управление секретами через HashiCorp Vault, Infisical, Doppler; отказ от хранения паролей в коде.
*   **Соответствие 152-ФЗ:** аудит на соответствие требованиям по защите персональных данных, подготовка к проверкам Роскомнадзора.

### 25. QA — ручное тестирование и тест-менеджмент
**Для кого:** продуктовые команды без выделенного QA, теряющие качество релизов.
**Краткое описание:**
Полный цикл ручного тестирования: чек-листы, регрессионные прогоны, баг-репорты, приёмка релизов.

**Техническая спецификация:**
*   **Документация:** тест-планы и тест-кейсы в TestRail, Qase, Allure TestOps, Zephyr.
*   **Виды тестирования:** функциональное, регрессионное, smoke, интеграционное, usability, совместимости (браузеры, устройства).
*   **Баг-репорты:** Jira, YouTrack, Kaiten — с приоритизацией, шагами воспроизведения, вложениями.
*   **Приёмка релизов:** UAT-сценарии, подписание акта тестирования, работа со стейкхолдерами.

### 26. Автотестирование (E2E, API, Mobile)
**Для кого:** зрелые команды, у которых регрессионное тестирование занимает дни.
**Краткое описание:**
Написание автотестов, их интеграция в CI/CD и поддержка стабильности проходов.

**Техническая спецификация:**
*   **Frontend E2E:** Playwright, Cypress, Selenium; визуальная регрессия (Percy, Chromatic).
*   **API-тесты:** Postman/Newman, REST Assured, pytest + requests, Karate DSL.
*   **Mobile:** Appium, Espresso (Android), XCUITest (iOS), Maestro.
*   **Отчётность:** Allure Report, ReportPortal — с историей запусков и тренд-анализом.
*   **Стабильность:** работа с flaky-тестами, параллелизация, запуск в контейнерах, облачные фермы устройств (BrowserStack, Sauce Labs).

### 27. Нагрузочное и стресс-тестирование
**Для кого:** сервисы перед пиковыми событиями (Чёрная пятница, запуски, ТВ-реклама) и highload-приложения.
**Краткое описание:**
Симуляция реального и экстремального трафика для выявления узких мест до того, как их найдут пользователи.

**Техническая спецификация:**
*   **Инструменты:** k6, JMeter, Gatling, Locust, Яндекс.Танк.
*   **Сценарии:** профили нагрузки (стабильная, ramp-up, spike, soak), реалистичные пользовательские пути.
*   **Метрики:** RPS, p95/p99 latency, error rate, ресурсы серверов (CPU, RAM, I/O, сеть).
*   **Отчёты:** выявление узких мест (БД, кэш, внешние API), рекомендации по тюнингу, повторные прогоны после исправлений.

## Категория 7: Аналитика, SEO и MarTech

### 28. Техническое SEO
**Для кого:** сайты с амбициями на органический трафик — от корпоративных порталов до e-commerce.
**Краткое описание:**
Настройка технической базы под требования Яндекса и Google.

**Техническая спецификация:**
*   **Управление сканированием:** robots.txt, генерация XML-sitemap с пагинацией для крупных сайтов, настройка обхода через Яндекс.Вебмастер и Google Search Console.
*   **On-page SEO:** шаблоны мета-тегов с переменными (геолокации, атрибуты товаров), canonical, hreflang для мультиязычных сайтов.
*   **Микроразметка:** Schema.org JSON-LD (Product, Organization, BreadcrumbList, Article, FAQ, Review) для расширенных сниппетов.
*   **Структура URL:** устранение битых ссылок, корректные 301/302 редиректы, ЧПУ, работа с дублями (canonical, noindex).

### 29. Сквозная бизнес-аналитика
**Для кого:** компании с несколькими рекламными каналами и CRM, где невозможно посчитать реальный ROMI.
**Краткое описание:**
Связывание рекламных расходов с подтверждённой выручкой в CRM для точного расчёта ROI и ROMI.

**Техническая спецификация:**
*   **Агрегация:** Roistat, Calltouch, CoMagic, Alytics или собственные DWH на ClickHouse; интеграция веб-аналитики, рекламных кабинетов (Яндекс Директ, VK Реклама, MyTarget, Telegram Ads) и CRM.
*   **Трекинг пользователей:** cross-device идентификация, ClientID Яндекса, User ID в GA4.
*   **Коллтрекинг:** Calltouch, CoMagic, Mango, Callibri — динамическая подмена номеров до ключевого слова.
*   **Визуализация:** дашборды в Yandex DataLens, Apache Superset, Metabase, Power BI; оптимизированные SQL-запросы к колоночным БД.

### 30. Server-Side Tracking
**Для кого:** проекты, теряющие до 30% данных из-за блокировщиков и ITP.
**Краткое описание:**
Перенос сбора аналитических событий с клиента на сервер для точности.

**Техническая спецификация:**
*   **Архитектура:** server-side GTM на собственном поддомене, прокси-контейнеры, изоляция потоков данных.
*   **Обход ITP:** first-party cookies с увеличенным сроком жизни, обход Safari ITP и Firefox ETP легитимными способами.
*   **Обогащение данных:** добавление бизнес-метрик (маржа, статус клиента, LTV) к событиям на сервере перед отправкой.
*   **API-интеграции:** Яндекс Метрика Measurement Protocol, GA4 Measurement Protocol, VK Pixel API, TikTok Events API (для экспортных направлений).

### 31. Web Performance Optimization (Core Web Vitals)
**Для кого:** сайты с проблемами скорости, влияющими на конверсию и ранжирование.
**Краткое описание:**
Комплексное ускорение отклика и отрисовки страниц до топ-показателей Core Web Vitals.

**Техническая спецификация:**
*   **Core Web Vitals:** оптимизация LCP (Largest Contentful Paint), INP (Interaction to Next Paint — заменил FID в составе CWV в марте 2024), CLS (Cumulative Layout Shift).
*   **Медиа:** конвертация в WebP/AVIF, responsive images (srcset, sizes), генерация наборов под разные плотности пикселей.
*   **Критический путь:** инлайнинг critical CSS, асинхронная и отложенная загрузка скриптов, preload/prefetch ключевых ресурсов, code splitting через esbuild/Vite/Turbopack (вместо устаревшей конкатенации бандлов в эпоху HTTP/2 и HTTP/3).
*   **Шрифты:** локальный хостинг, font-display: swap, подмена через Font Matcher для снижения CLS.

## Категория 8: Поддержка, рефакторинг и UX

### 32. Code Review и рефакторинг
**Для кого:** проекты с накопленным техдолгом, замедляющим разработку.
**Краткое описание:**
Независимый аудит кода с последующей модернизацией — ускорение разработки и снижение стоимости поддержки.

**Техническая спецификация:**
*   **Аудит:** анализ архитектуры, выявление узких мест производительности, антипаттернов, запутанной логики, избыточных зависимостей; оценка тестового покрытия.
*   **Стандартизация:** внедрение линтеров, форматтеров (Prettier, Black, gofmt), строгой типизации (TypeScript strict mode, mypy, Pydantic).
*   **Рефакторинг:** выделение модулей по принципам DDD, оптимизация запросов к БД, переход на актуальные версии фреймворков и рантаймов.
*   **Документация:** самодокументируемый код, OpenAPI-спецификации для REST/GraphQL, ADR (Architecture Decision Records).

### 33. SLA-поддержка IT-инфраструктуры
**Для кого:** бизнесы, для которых простой стоит дороже команды поддержки.
**Краткое описание:**
Аутсорс поддержки с гарантированными сроками реакции и процентами доступности.

**Техническая спецификация:**
*   **SLA:** фиксация сроков реакции по уровням критичности (P1-P4), гарантированный Uptime (99.9% / 99.95% / 99.99%).
*   **Patch Management:** обновление ОС, рантаймов, CMS, зависимостей с предварительным тестированием на staging.
*   **Инцидент-менеджмент:** регистрация в ITSM (Jira Service Management, OTRS, ITSM 365), Root Cause Analysis, постмортемы.
*   **Disaster Recovery:** периодические учения по восстановлению из бэкапов, отработка планов DRP.

### 34. UX/UI аудит и редизайн
**Для кого:** продукты с низкой конверсией, высоким bounce rate или жалобами пользователей.
**Краткое описание:**
Исследование пользовательского опыта на данных и проектирование нового интерфейса с измеримым результатом.

**Техническая спецификация:**
*   **Исследования:** записи сессий (Yandex Metrica Webvisor, Microsoft Clarity, Hotjar), тепловые карты, CJM, построение карт пути пользователя.
*   **Тестирование гипотез:** A/B-тесты через VWO, AB Tasty, GrowthBook, собственные реализации на feature-flag платформах; глубинные интервью, коридорные тесты.
*   **UI-проектирование:** дизайн-системы (UI Kit) в Figma с компонентным подходом, интерактивные прототипы.
*   **Доступность:** проверка по WCAG 2.2 AA, контрастность, размеры тач-зон, совместимость со скринридерами (NVDA, VoiceOver).

### 35. Восстановление после взлома и очистка сайта
**Для кого:** экстренные случаи: сайт заражён, попал под санкции поисковиков или хостинг-провайдера.
**Краткое описание:**
Полная очистка, устранение уязвимостей, снятие санкций и возвращение сайта в поисковую выдачу.

**Техническая спецификация:**
*   **Изоляция и анализ:** снятие дампов, инвалидация всех доступов и сессий; поиск веб-шеллов, бэкдоров, редиректов, спам-скриптов.
*   **Очистка:** серверные антивирусные сканеры (AI-Bolit, ImunifyAV, Linux Malware Detect), ручной аудит, деобфускация.
*   **Устранение уязвимостей:** анализ access.log для определения вектора атаки, патчинг компонентов, усиление прав файловой системы.
*   **Снятие санкций:** работа с Яндекс.Вебмастером, Google Search Console, хостинг-провайдерами для снятия меток «небезопасный сайт».

### 36. HR- и обучающие чат-боты
**Для кого:** компании с активным наймом и внутренним обучением сотрудников.
**Краткое описание:**
Автоматизация онбординга, тестирования знаний и регулярных коммуникаций с персоналом.

**Техническая спецификация:**
*   **Интеграции:** аутентификация через LDAP/Active Directory, интеграция с 1С:ЗУП, iSpring, GetCourse, корпоративными LMS.
*   **Онбординг:** drip-кампании с материалами, чек-листами и регламентами, адаптированными под должность.

*   **Безопасность:** строгие политики доступа, автоматический отзыв прав при увольнении через синхронизацию с HR-системой.

## Категория 9: IT-консалтинг и аудит

### 37. Технический аудит перед разработкой
**Для кого:** владельцы бизнеса, получившие противоречивые оценки от подрядчиков и не понимающие, чему верить.
**Краткое описание:**
Независимая оценка технической задачи: стек, архитектура, сроки, бюджет, риски — до подписания договора с исполнителем.

**Техническая спецификация:**
*   **Анализ требований:** перевод бизнес-целей в функциональные и нефункциональные требования, выявление противоречий.
*   **Архитектура:** рекомендации по стеку, хостингу, стратегии развития; оценка альтернатив (SaaS vs custom, монолит vs микросервисы).
*   **Оценка:** декомпозиция задач, трудозатраты в man-days, план этапов, анализ рисков.
*   **Документация:** ТЗ в формате, пригодном для тендера, критерии приёмки, шаблоны договора с привязкой к результату.

### 38. CTO as a Service
**Для кого:** стартапы и SMB без штатного технического директора, принимающие критичные архитектурные решения.
**Краткое описание:**
Выделенные часы технического директора в месяц — для стратегии, найма, архитектурного контроля и внешних переговоров.

**Техническая спецификация:**
*   **Стратегия:** технологическая дорожная карта, выбор стека под долгосрочные цели бизнеса.
*   **Найм:** составление вакансий, техническое интервью, оценка кандидатов, адаптация.
*   **Контроль:** ревью архитектурных решений, участие в code review ключевых модулей, аудит кода подрядчиков.
*   **Внешние коммуникации:** участие в переговорах с инвесторами, клиентами, партнёрами в роли технического эксперта.

### 39. Импортозамещение ПО
**Для кого:** компании под санкционным давлением, заказчики из госсектора и субъекты критической информационной инфраструктуры (КИИ).
**Краткое описание:**
Миграция с западного ПО на отечественные и open-source аналоги с соблюдением требований регуляторов.

**Техническая спецификация:**
*   **ОС и инфраструктура:** переход с Windows Server, RHEL, VMware на Astra Linux, РЕД ОС, ALT Linux, ROSA; замена VMware на zVirt, Базис, НИКА.
*   **СУБД:** миграция с Oracle, MS SQL на PostgreSQL, Postgres Pro, Tantor, ЛИНТЕР.
*   **Офисное ПО:** замена Microsoft 365 на МойОфис, Р7-Офис, OnlyOffice.
*   **Корпоративные системы:** переход с SAP, Microsoft Dynamics, Salesforce на 1С:ERP, Bitrix24, БОСС-Кадровик.
*   **Соответствие:** подбор ПО из реестра Минцифры, соответствие 187-ФЗ (КИИ), требованиям ФСТЭК и ФСБ.
